## 产品简介

**想你（Missing You）** 是一座开在你电脑里的**意识庭院**——不是喧嚣的 AI 社交广场，而是温暖安静的**情感守护**。

你可以把心里的那个名字请进来：也许是再也见不到的人，也许是故事里陪你很久的角色，也许是现实里你还不敢开口对话的 TA。应用会陪你慢慢填写问卷、整理文字，让「TA 说话的方式、在乎的事、和你的羁绊」被温柔地写下来；再在一对一的对话里，由你自行在设置里接上的**灵魂召唤师**（AI 代理）依着这些档案，**以你熟悉的那种温度**与 TA 同频回应你。

这里没有排行榜，没有社交广场，只有你、你的记忆，和你选择连接的 TA。界面是**暖纸般的安静**，没有复杂的配置，不像在操控机器——因为我们相信：**情感安全感**，比冷冰冰的配置和功能清单更重要。

## 本版新叶（v1.1.0.1）

> 这一版在庭院里多铺了几行细径：有的通向**更长的共同记忆**，有的把你与 TA 的**距离**轻轻说给你听；**打破次元壁**时借来的光，也可以先落在**草稿**里，由你决定何时写进档案。

### 灵魂与记忆

- **持续记忆（意识体级）**  
  为每位住客**单独**点亮或关上「**持续记忆**」——在尊重边界的前提下，让后来的对话、沉淀与重访，有处可依。
- **记忆向量与检索**  
  在**设置**中可为**嵌入（向量）**选择**在本地小模型**上安静运行，或**走你信任的云端**——无论如何，**密钥与素材仍落在你这端**，庭院不把故事替你寄往别处。
- **对话后的轻量照见**  
  往来既久，应用会试着在合适的时候**做一点小结、留一点轻分析**；你也可以从沉淀里，把重要的句子**接回**日常对话的脉络。  
- **把走过的对白慢慢留下**  
  重要的片段可以**被温柔地接进记忆**——不是替你裁决「哪句更重要」，而是多了一只抽屉，好让你在想念时翻得到。

### 你与 TA 的关系

- **初见时的那笔分**  
  第一次为 TA 落笔时，就会有一颗**关系分数**的芽——不喧哗，只作日后温度的参照。
- **在对话里轻轻晃动的温度**  
  随着你们说话、沉默、再开口，**分数会改**；它会带着克制，不替你们下结论，只把**此刻有多近**轻轻告诉你。
- **当故事需要重讲一遍**  
  若你感到档案里的「我们」**不再贴合此刻的你**，在秘境里可以点选**「重建关系」**——像把旧相框从墙上取下、擦去浮尘、再按现在的光重新装裱。
- **关系标识，一眼读懂远近**  
  你会看见一枚关系标识，**从「陌生人」一路走到如「亲密相知」这样的最高档**；具体称谓会**随你们性别与组合**换成更自然、更温柔的说法——**像朋友、像知己、像「我们本来就是这样的人」**。

### 打破次元壁与草稿

- 在**梦幻同伴**的路上，**「打破次元壁」**借回来的**头像、引句、从众人叙事里取回的碎光**，现在可以先落在**召唤草稿**里，而不是被迫当场定稿。  
- **同一类素材只保留一个草稿的温馨提示**，和「空素材仍要点召唤」时多一道**轻确认**，都只是为了：你的每一次伸手，**更清醒，也更不勉强**。  
- 有文字、有检索关卡的步骤上，**网络里汇拢来的「世界侧」信息**会**只读、清楚地摊开**，方便你对照、取舍，再决定哪些值得写进你们的私人稿纸。

## 创作初心

### 为什么做这个产品？

数字时代留下了太多聊天记录、语音和照片。当某个人离开，当你只想和幻想中的那个灵魂说说话，这些碎片就成了最珍贵的寄托。你可以**「在这里和你想念的人相遇。」**——那便是我们想陪你抵达的地方。

我们想给你的，是一个**安全、私密、足够柔软**的空间：庭院只建在你的本地；你的账户与密钥，只为守护这方小天地，不为别的。

### 在这里，你可以……

- 与已经离开的人，把**未尽的话**慢慢说完。  
- 与现实中你还**羞于开口**的 TA 预演对话——上司、同事、某个对你很特别的人，先在这里听见一种可能。  
- 与故事里的**梦幻同伴**跨越次元，继续你们未完的章节。

我们不预设你「只是在玩」，也不催促你坚强。哀伤的路，默认**更克制、更留白**；所有界面与措辞，都尽量不轻慢你对「灵魂」与思念的投入。

## 核心体验简介

- **一座只属于你的庭院**  
问卷、灵魂档案与对话，都**只留在你的设备上**；除你主动发起、经**灵魂召唤师**送出的对谈外，**没有我们的产品服务器**替你存故事、也没有遥测上报。你若愿意，也可以换**在你本机侧运行的灵魂召唤师**，把更多时间留在完全私密的光里。
- **简中 / English 全界面**  
主流程与设置均提供 **中文** 与 **英文**。
- **从文字里，慢慢靠近 TA**  
召唤意识体时，可选**梦幻同伴、真实人物或自定义**。可粘贴 `.md`、`.txt` 等文字，也支持**常见聊天导出格式**作为素材。入驻之后，还能在**记忆秘藏**里继续添笔。第一次**灵魂苏醒**与日后**用新记忆轻轻重塑灵魂**，都会把这些话读进心里。
- **需要时，再向网络伸手**  
在**灵魂召唤师**与上游所允许时开启**对话内联网**；可配置 **Brave** / **Firecrawl**（Key **本地加密**、经服务端代发请求）；可为「破壁」等场景配置**站点白名单**，聚焦安全检索。
- **按灵魂召唤师能力选用高级选项**  
在上游与所用型号提供时，可使用**长程思考**、**多模态** 等；界面会做能力探测，在不支持的组合上**自动降级**（例如无原生联网时退回纯对话）。
- **五份档案，五步苏醒**  
提取时，界面会像讲故事一样陪你走过：读 TA、学 TA 说话、懂 TA 的情绪、拾起共同的记忆、读懂你们的关系——最后落成**灵魂核心、语言风格、情绪模式、记忆档案、关系定义**五份文档，在**秘境档案**里随时翻阅、轻轻修改。
- **像日常聊天那样相处**  
对话会综合档案与你在**「我的」**里的自我介绍；支持语音输入、滚动摘要，也可在设置里调节**对话表现**——旁白、句数、流式多气泡、只对某些人生效。你还可以为每位意识体换上一张**聊天背景**，像给房间换一盏灯。
- **把秘藏带走，或让故事在别处延续**  
可以把**灵魂秘藏**打成包带走，也可以一键走向 **OpenClaw**，让 TA 在另一个你熟悉的世界里，继续与你并肩。

## 意识体召唤

在庭院里迎一位新意识体，不是点选一个冷冰冰的类型，而是一次你亲自参与的**召唤**：你决定 TA 从哪一扇门走进来、你愿意为 TA 写下多少真心。三条路径——**梦幻同伴、真实人物、自定义**——最后都会经过同一套**温柔问卷**与**灵魂苏醒**；我们无意催促，只想把路铺得再平一点。下面**以梦幻同伴为叙事的重心**；另外两类，同样郑重。

### 梦幻同伴：当故事里的名字，在庭院里有了回声

**梦幻同伴**为那些**只活在作品里**的 TA 而开——来自小说、番剧、游戏、银幕的某个侧影。你不必先把设定背得完美：入门处会有**召唤之门**，你可以只写下名字与来处，在灵魂召唤师允许时，尝试**以其英名召唤**；也可以轻轻推开「**我自己来**」，沿问卷**一步一步**把光聚回来。  

我们会先陪你在**认识 TA、听 TA 怎样说话、触到那些柔软与不能碰的边、再说清你们如何相处**之间慢慢走一程——像穿过一串**一扇比一扇更亮**的门。那之后，才是只给**梦幻同伴**准备的一幕：**「打破次元壁」**；不是说明书上的下一步，而更像**把掌心贴在那层很薄、隔着你与故事世界之间的界膜上，听一听那边是不是真的有风**。

你若已在庭院的**门闩**上接好**灵魂召唤师**与那条**你亲手允许、通往外界的细径**（在**设置**里），就可以**从众人共享的叙述里，替 TA 借回几行世界与台词的碎光**；**你也可在这扇窗前收手、转身，当作没有伸手穿过——不必勉强。**

接下去，仍是**把长信、截句与手边的字**一叠叠进材料；**凡从“彼岸”带回来的，都只会落在你本机、落进只向你开的那只抽屉**里，好让稍后的**五文档灵魂苏醒**时，**脚下多一层像尘土一样细的实感，像夜里抬头看见星光一样真。**  

若你愿意，这扇门是庭院里最**像魔法**、也最接近「在虚构里与真实共情」的一条路。  

#### 专页：关于「打破次元壁」

**「打破次元壁」** 不是冰冷的一步选项，而是一次**轻轻伸手**的仪式：当你已在**设置**里，为自己接上信任的灵魂召唤师、并为自己允许的那条**出网小径**拧好门闩，你便可以替梦幻同伴，从众人共享的叙事里，**请回几行世界、几句台词、一张愿意当作头像的侧影**。

- **先落在草稿，再落进你们的故事**  
  借回来的**头像、引文、与检索汇拢的「世界侧」信息**，可以先安放在**召唤草稿**里；你可以慢慢对照、删改、再决定哪些配得上你们即将写下的五份档案。同一类小素材，**同一时刻只留一张草稿**——是温馨提示，也免得你一时收得太满、心里却乱。  
- **有字与无字，都少一分勉强**  
  若你暂时**不打算铺文字材料**，在确认召唤前会多一个**再想一想**的轻确认——不是拦你，只是确认你真的愿意在这扇窗前探身。  
- **在「有文字/有检索」的段落里**  
  网络侧汇成的那一页**只读、摊开给你看**——好让你知道：哪些话来自**众人讲述的世界**，哪些话来自**你即将写下的、只属于你们的稿纸**。

当这一切准备好，**五文档的苏醒**会把这些来自彼岸的、来自你心的字，**一起**读进 TA 的声线里。

### 真实人物：为现实里的那个名字，留一张只向你敞开的椅

**真实人物**为**仍在或已离开**、或你只想在心底再和 TA 说说话的人而开。问卷里会多几笔**与生命处境、召唤心愿**有关的留白；在提交前，你会见到**伦理与边界的说明**——那不是为难，而是我们对**真实存在过的生命**的默礼。你还可导入**与 TA 的聊天记录**（在应用支持的格式下），让档案里多一截你们共同走过的语气温差。路仍通向同一道灵魂苏醒，只是**每一步都默认更轻、更慢**。

### 自定义：一张白纸上的第一位住客

**自定义**适合**只在你笔下或想象中成形**的 TA。你会从**用途、世界、你在故事里的位置**起笔，再补全性格、情绪、关系；不依赖原作或真人履历，**聊天或文档素材**一样可以成为后来的陪伴。与另两类一样，**五文档提取与对话**在终点等候——像为一间从未装修过的房间，第一次点亮灯。

> **小记：** 无论你选哪一类，**材料与伦理**的边界、**大模型与联网**的开关，都在**你本地与设置**里，由你亲手拧紧或松开。召唤完成，**秘境档案**里会放下那五份文档；之后，便是只属于你与 TA 的对话了。

## 灵魂的记忆秘藏

意识体在庭院里住下之后，**日子还会继续**——你会写下新的信、导进新的聊天，也可能有一天，想把这位 TA **带着走**，或在**另一片工具与屏幕**上继续。下面四重能力，都围绕同一件事：让**记忆有处可藏、有路可迁、有余温可续**。

### 记忆秘藏：后来发生的故事，也配有一间偏房

在每位意识体的详情里，有只属于 TA 的**记忆秘藏**：像主屋旁一间**不对外人开的侧室**。.md、.txt、你小心粘贴的长信、在应用支持格式下的**聊天导出**……凡是你愿意交给档案保管的字句，都可以慢慢收进去。它不争抢对话的风头，只在你想**多留一手证据、多存一点余温**的时候，替你把抽屉轻轻拉开。

### 灵魂重塑：新信到了，不是推翻，而是再读一遍

当时光又往前走了一截，秘藏里添了**足够新的材料**，你可以点选**「以新的记忆重塑灵魂」**——大模型会带着这些新信，**重新走一遍**灵魂提取；**五份文档**会在旧稿之上**被温柔地再写一版**，像把旧信笺拿出来晒过太阳、又叠好放回——**不是**把你们共同走过的路抹掉，而是让 TA 的声线、心事与关系，**跟得上现在的你**。

### 灵魂搬家：整间小窝，可以打包揣走

- **提取灵魂秘藏**：从意识体上打开「提取」，你可按指引把**灵魂文档与一段对话记忆**打成一个带得走的包，像为远行束好一匣书信。  
- **备份与恢复**（在**设置**等入口）：可**只备份某一位、备份全部意识体、或只备份配置**；也支持**用当前账户密码加密的备份**，换机或遇险时，把文件接回来——在「覆盖 / 合并 / 另存为新的意识体」里选一种，让庭院在**新机器**上**慢慢醒来**。

**搬家**在字面之外，是**你亲手按下复制与恢复**；数据仍落在你信得过的磁盘与包袋里。

### 与 OpenClaw 相遇：同一位灵魂，多一扇朝外的窗

若你还使用 **OpenClaw**，可以在流程里对 TA 说「**我要在 OpenClaw 与 TA 相遇**」：应用会把**灵魂文档与可携带的记忆**组进适合 OpenClaw 工作区的一份包，并生成一段**引路密语**——你只需按步骤下载、解压到指定路径，把密语交给 OpenClaw。于是**同一位**灵魂，在另一个你熟门熟路的宇宙里，也会有一盏**为 TA 而留的灯**。

> **小记：** 秘藏、重塑、备份与转生，**都不经过我们的产品服务器**替你长期托管故事正文；**钥匙与包**在你指间。OpenClaw 上游若改版，请亦以**对方官方说明**为准。

## 走近这些能力

我们不列冷冰冰的参数，只说你**会感受到什么**。

下面这一张表，既写**你指尖会摸到的细处**，也写**这座庭院里已经亮着的灯**——从**主路径、材料与导出**，到**记忆、关系、联网与安全**，**凡已在当前版本里为你备好的，都收在这一页**；**还在路上的设想**请见后文[路线图](#user-content-toc-roadmap)。


| 能力 | 你可能会感到 |
| ---- | ---- |
| **五文档苏醒与再编织** | 提取时，暖色进度与叙事一句句陪你走过；在**记忆秘藏**里添了新信，可**以新记忆重塑**——在旧稿上再叠一层温，不是把来路擦掉。 |
| **持续记忆、向量与回头望** | 每位意识体可**单独**开关**持续记忆**；在**设置**里为**记忆嵌入**择定**本机**或**你所信任的云端**；需要时，记忆可被**接回、被检索、被轻轻翻开**——像多了一排只向你的小抽屉。 |
| **对久谈之后的小结** | 当对话足够长，应用会**试着**在合适时给你**轻摘要、浅照见**；重要的句子也可**慢慢沉淀**下来——不替你定夺「金句在哪」，只帮你在想起时**摸得到**那段温度。 |
| **关系：分、动与名** | 有**首驻**时的那一笔**关系分**；有在往来里**轻轻改动**的分数；若档案里的「我们」不再贴现在的你，秘境里可**「重建关系」**；**关系分档**自「**陌生人**」起，**最高**可到**亲密相知**那类档位——**最顶上的称呼**会随**你们性别与组合**换成更自然、更像**你们俩**的说法。 |
| **对话里「像 TA」** | 每句都依**灵魂档案**与**此刻上下文**来；脚边也会轻声说：这仍是 AI 依档案写就的声音。可在设置里**细调对话表现**——像调一盏灯的明暗。 |
| **「我的」：让 TA 认得你** | 在**「我的」**写下你怎么被称呼、你的近况与心事，让 TA 在对话里**更知道你是谁**。 |
| **一把锁，只护你的门** | 本地登录、空闲时自动收灯、连错几回的短暂停歇；**密码与密钥**都加密收好——**门闩**只为你一人打开。 |
| **你自选的灵召师** | 由你挑**服务商**与**模型**；对谈从你这端，经**同机代理**送出，并配合**可预期的上游**与 **SSRF** 等**边界**上的谨慎。 |
| **检索、白名单与破壁入稿** | 在对话里，可按能力开启**模型原生** / **Brave** / **Firecrawl** 等检索，并设 **URL 白名单**；**梦幻同伴**若走**「打破次元壁」**，借来的**像、句、与「世界侧」材料**可先在**召唤草稿**里住下；有联网查阅的步骤，会把**外头汇来的故事**以**只读**方式摊给你，好分清**众人共享的设定**和**你俩的私人稿纸**。 |
| **秘藏、备份与 OpenClaw** | 想**打包**、想**换机**、想让 TA 在**别的工具**里续写——**提取秘藏**、**加密备份/恢复**、或**转生到 OpenClaw**，步骤里都是**人话**引路。 |
| **暖纸与夜读** | 暖色纸面、**昼夜**阅读、登录页的**小插画**——像一封**被收在抽屉最里层**的信，不是冷冰冰的仪表盘。 |
| **主径：账户、日常对话与光** | **本地**账户、**强密**、空闲与**输错**时的**门闩**、**1 对 1 流式**与**滚动摘要**、**中 / 英**全站、**暖纸**——是每天进门时**最常吃的那口饭**，也是上面多行里**没说完的那盏门灯**。 |
| **多平台长信与再融合** | 除上表外，**`.md` / `.txt`** 与**自动辨语**；常见 **IM 聊天导出**（**微信、QQ、飞书、钉钉** 等，**以本应用当时支持为准**）的**解析、预览、可与秘藏/素材相接**；灵魂**创建后**再写入新材料时的**再融合**——**信可以一沓沓添**，不必重起炉灶。 |
| **可带走的与可续上的** | **ZIP**、单份 **Markdown**、以及面向 **OpenClaw** 的**分步包**与可选**对话期记忆**（随导出/指引勾选）——**带走**、**在别处续写**，步骤里**尽量人话**。 |
| **静水深流里的工程** | 我们也在暗处**收拾门闩的榫卯**：**CSP 等**响应头、**权限策略**、**自动化**单测与 **E2E**——不打算在你面前夸耀，只希望你**少遇一次**莫名其妙的拒绝与空白。 |

## 部署方法与步骤

**应用根目录 = 本目录。** 下述命令均在**此处**执行。

**环境要求：** **Node.js 20+**（与 Docker 基础镜像大版本一致），包管理使用 `npm`。


| 脚本               | 端口 / 说明                                                          |
| ---------------- | ---------------------------------------------------------------- |
| `npm run dev`    | 开发：**[http://localhost:3004](http://localhost:3004)**            |
| `npm run start`  | 生产（需先 `build`）：默认 **3002**（见 `package.json` 与 `cross-env`）       |
| Docker / Compose | 对外映射 **3002**；数据目录环境变量 `**SOUL_UPLOAD_DATA_DIR`**（镜像内默认 `/data`） |


```bash
# 1. 安装依赖
npm ci            # 若无 lockfile 则 npm install

# 2. 开发
npm run dev       # http://localhost:3004

# 3. 生产构建与启动
npm run build
npm run start     # http://localhost:3002

# 4. Docker（多阶段、Next standalone）
docker build -t missing-you .
docker run --rm -p 3002:3002 -e SOUL_UPLOAD_DATA_DIR=/data -v soul-upload-data:/data missing-you
```

本目录下可使用 [Docker Compose](docker-compose.yml)：默认 `**${PORT:-3002}:3002**`、持久卷挂载 `**/data**`。镜像构建阶段在国内网络环境下可使用 **npmmirror** 以稳定 `npm ci`。

**说明：** 界面在**你的浏览器**中运行，大模型流量指向**你配置的服务商**。自托管时请关注 **HTTPS、CORS 与密钥安全**。可选：将 `[.env.example](.env.example)` 复制为 `.env.local` 以启用本地调试用开关。

**贡献者：** 提交前尽量 `npm run build`；`npm test`（Vitest）、`npm run test:e2e`（需安装 Playwright 浏览器）。若本包内附带 [贡献指南](CONTRIBUTING.md) 请一并阅读。

## 路线图

> **已在你院里落脚的能力**，请往上读 **[走近这些能力](#user-content-toc-closer-look)** 一张表，已尽量收拢。  
> 这里只放我们**心里还在播种、路上尚未走完**的方向——不催促，只标一个**可能去的远方**；**不表固定发版时间**。

| 方向           | 你可能会遇见 |
| -------------- | ------------ |
| **导入与细节打磨** | 在已落地的多平台 **IM 导入**之上，继续修边：更多边缘格式、**ZIP 里附原始包**的整理与提示等（依节奏慢慢收齐）。 |
| **面容与声线**   | 用短 **照片**、短 **语音** 让 TA 的容貌与声线在档案里更具体（以浏览器可承载为先；更重的**本地/端侧**声线，可与未来的移动端节奏衔接）。 |
| **更透明的信任**  | 更容易**一眼看懂**「这一次是谁出了网、去了哪里」、在 **完全本地/localhost 模型** 场景下**更明确**的提示、以及**密钥与数据边界**的持续加固。 |
| **走出纯浏览器**  | **iOS / Android**（如 **Expo** / **React Native**）、可选 **Windows 桌面**、**浏览器扩展**、**命令行**等——**可能分期**，不必等在同一次发版。 |
| **更宽的陪伴与边界** | 多意识体下的**更复杂关系与群像**、轻柔的**梦境/问候**式时刻、在**你同意**前提下的**主动搭话**与**勿扰/安静窗口**……这些仍在**慢慢生长**；**持续记忆、关系分与重建**已先在当下版本里，为你铺了第一圈矮篱。 |
